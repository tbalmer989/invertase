# Decisions Log

## 1. Design & Scope

Firstly I looked at the specification I then had a look at the Genkit Github as I've never used Genkit before.

I then chose to develop the plugin in Python as it's the language I'm most familiar with as well as the development tools for (such pipenv). Additionally I figured it's readability would be a benefit especially given the short development time frame

Otherwise my tooling has been fairly standard, using VScode as an IDE and WSL as Python is found natively on Linux and historically I've written most of my code on Linux systems. Another advantage of VSCode was the inclusion of Co-pilot which made development quicker (as will be seen in the next section)

As the brief was not clear on the specifics types of URL that the plugin was expected to be used on (i.e. social media, news sites or Youtube) I decided to focus only on text heavy sites and selected five sites (see [Articles](#articles)) for user driven tests, as I wasn't sure I'd have the time to generate suitable complex automated tests.

These sources were chosen as the layout of these pages are varied, they're text heavy, and most have adverts on them. I did consider using Facebook or Youtube (as the plugin can be used against any site but given the short timeframe and the added complexity to extract the comments probably would force this out of brief). In hindsight this is probably what caused the issues with the Reddit and Stack Overflow links.

The articles chosen where from the front page of their respective websites on the 20-05-2026

## 2. AI Orchestration

After reading the specification document and having a brief look at the Genkit Github repository. I quickly went through the Getting Started part of the documentation as I wanted a basic understanding before generating code.

After completing this and setting my environment up I used Copilot to generated a base Genkit project in a second repository to use the plugin with the following prompt:

    In the genkit-base directory can you please generate a base genkit application that uses the plugin found in the invertase directory?

This created the code, tests and installed required pip files. With the base Genkit application I re-read the specification document to craft the following prompt:

    Build a Genkit plugin called ‘genkitx-readability’ using Python that registers a tool for extracting clean, structured content from web URLs.
    Given a URL, the tool should return the page's main content stripped of navigation, ads, scripts, and boilerplate. The output should be structured (not just a raw string) and useful as input to an LLM.

    Also include an agent skill in the repo that helps an AI coding assistant understand how to maintain and extend this plugin.

As part of the generation this first iteration of the code was validated using  tests generated by Copilot. As mentioned in the brief I wanted to confirm the tests worked against a more complex site so I used the BBC [article](#articles) to get an output.
As some social media/get in touch text was still in the output I used the following prompt to add the option to remove it:

    Can you modify the plugin so that it will add the option exclude the following:

    1. related content/topics headers and the links/text under them
    2.Invitations to get in touch/comment

I could see scenarios where this content could be wanted which is why added it as an optional flag. A second prompt was then used to add the tests. After implementation Co-pilot had suggestions to cover edge cases such as pluralisation and foreign languages sites.

I extended the functionality of the plugin so that the output could be saved either to text or JSON files.

        Can you add functionality to allow exporting the output to txt files or json files if JSON flag is set to true please?

This also added the the tests for the functionality.

## 4. Agent Skill

An agent skill was included at the beginning of this process. However its development was largely separate to the plugin's.
I'm new to the use agentic AI not familiar with agent skills.
To better understand agent skills I've looked at both the documentation on GitHub as well as specifically asking Google's Gemini about what skills it should have. I decided that the following skills would be useful:

1. Abstract Syntax Tree (AST) & Node-Level Traversal Skills
   - A true "Reader View," the agent cannot rely on basic regex or string splitting. It must parse the HTML into a stateful, mutable tree and programmatically assess it.
2. Token-Optimal Schema Synthesis
    - Standard web scrapers emit raw markdown or HTML this can be token expensive so by using Type-Safe serialisation and JSON schemas we can have an agent build an LLM-focused plugin with an understanding of token economics and structural inference.
3. Idiomatic GenKit Runtime Mapping
    - The agent needs to have an understanding of the Genkit lifecycle, as to generate specific code.
4. Headless Orchestration & Connection Resiliency
    -To bypass defensive infrastructure its needs to be able to spoof headers, rotate user agents. Also as most websites are now dynamic and ability to intercept client side rendered data the agent needs to be included
5. Automated Regression & Fixture Testing
    - The structure of websites changes regularly. Being able to build a suite of mock files that reflect different layouts allows the agent to evaluate the consistency of the output
